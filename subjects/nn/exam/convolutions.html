Convolutions

Why FC layers fail on images, and how local connectivity, learnable filters, padding, stride, receptive fields and pooling build up a CNN.

1 Why FC layers fail on images

  • A 224×224×3 image = 150,528 inputs. One FC layer to 1000 neurons = ~150.5M weights — and that's ONE layer.
  • Wasteful, overfits instantly, slow, memory-hungry.
  • Core issue: FC treats every pixel as equally related to every neuron → it ignores spatial structure.
  • Key fact: nearby pixels are correlated (sky is blue in patches); distant pixels are usually unrelated.

2 Local connectivity

  • Solution: connect each neuron to only a small local patch of the input, not the whole image.
  • Same job, drastically fewer parameters — each neuron sees only a small region.
  • Intuition (Where's Waldo): your eye scans local patches for a pattern, then recognises it anywhere in the image.

3 Filters / kernels

  • A filter (kernel) is a small grid of learnable weights (e.g. 3×3).
  • Operation = slide, multiply, sum, repeat: at each position, element-wise multiply with the patch then sum → one output value.
  • A filter responds (large output) where it finds its pattern; uniform regions → output ≈ 0; matching edges → large output.
  • Examples: vertical-edge filter [1 0 -1; 1 0 -1; 1 0 -1]; horizontal-edge filter [1 1 1; 0 0 0; -1 -1 -1].

4 Feature maps & dimensions

  • 1 filter → 1 feature map (detects one pattern). N filters → N feature maps. Typical: 32, 64, 128, 256 per layer.
  • Filter depth always equals the number of input channels.
ComponentDimensions
InputH × W × C
One filterF × F × C (spans all input channels)
N filters → outputH′ × W′ × N

e.g. 224×224×3 with 64 filters → 224×224×64.

5 Padding & stride

  • Shrinking problem: each conv layer shrinks spatial dims (5×5 → 3×3 → 1×1 → gone). Fix = padding.
  • Valid padding (no pad): output shrinks, e.g. 5×5 with 3×3 → 3×3.
  • Same padding (pad=1 for 3×3): ring of zeros around the border → output = input size (5×5 → 5×5). Most common.
  • Stride = how far the filter jumps. Stride 1 = every position (default); stride 2 = skip every other → output roughly halves.
  • Stride > 1 is used for downsampling — a modern alternative to pooling.

6 Output-size formula

Must-know for exam 
Output = (W - F + 2P) / S + 1
W = input size, F = filter size, P = padding, S = stride. Apply per spatial dimension; floor the result.

Rule of thumb: for "same" padding with stride 1, set P = ⌊F/2⌋.

7 Parameter sharing

  • Parameter sharing: the same filter weights are used at every spatial location.
  • Why: a vertical edge looks the same top-left or bottom-right → no need to learn a separate detector per position.
  • Result: one 3×3 filter = 9 parameters, reused at all positions; FC equivalent ≈ 150M.
  • Makes sense for images (a pattern can appear anywhere) but NOT for tabular data, where each feature has a distinct meaning.

8 Receptive field

  • Receptive field (RF): the region of the original input that one neuron "sees".
  • Stacking 3×3 convs (stride 1): 1 layer → 3×3, 2 layers → 5×5, 3 layers → 7×7. Each layer adds context.
  • Formula: RF = 1 + L·(F - 1) (L = layers, F = filter size).
  • Hierarchy of features: early layers → edges/colors (small RF); middle → textures/parts; deep → objects/scenes (large RF). This is why we stack conv layers.

9 Pooling

  • Pooling downsamples and summarizes. Max pooling (2×2, stride 2): take the max in each window → 4×4 becomes 2×2.
  • Idea: "is there a strong feature somewhere in this region? I don't care exactly where."
  • Three wins: (1) reduces spatial size / compute, (2) translation invariance (small shifts → same output), (3) increases receptive field.
  • Types: max pooling (classic), average pooling, global average pool (end of network).
Must-know tradeoff Pooling discards spatial precision — you know a feature exists but lose exactly where. Hurts pixel-precise tasks (segmentation, super-resolution).

★ Likely exam questions

Q1. A 128×128×3 image into an FC layer with 256 neurons — how many weights? Why a problem?

128·128·3·256 = 12,582,912 weights. Overfits on small data, computationally expensive, and ignores spatial structure.

Q2. Input 13×13, filter 5×5, padding 2, stride 2 — output size?

(13 - 5 + 2·2)/2 + 1 = 12/2 + 1 = 7 → 7×7.

Q3. Why does parameter sharing make sense for images but not for tabular features?

A pattern (e.g. a vertical edge) can appear anywhere in an image, so one shared filter works everywhere. In tabular data each feature (age, income) means something different — sharing weights across them is nonsensical.

Q4. Receptive field of two stacked 3×3 convs (stride 1)? Three layers?

Two layers → 5×5, three layers → 7×7. Each 3×3 layer adds 2 pixels of context. Formula: RF = 1 + L·(F − 1).

Q5. Main tradeoff of max pooling, and when does it hurt?

It discards exact spatial location — you know a feature exists in a region but not precisely where. Hurts pixel-level tasks like segmentation or super-resolution.

Q6. State the output-size formula and give "same" padding for a 3×3, stride-1 conv.

Output = (W − F + 2P)/S + 1. For "same" output with stride 1, P = ⌊F/2⌋ = 1 for a 3×3 filter.
